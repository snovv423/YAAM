# Telegram-бот ресторана — следующий шаг

Ещё не реализован. По архиву (`docs/YAAM-polnyy-arhiv-v6.pdf`, часть 5) бот должен:

- принимать/отклонять заказ, выставлять время готовки;
- двигать статус: Принят → Готовится → Передал курьеру → Доставлен;
- управлять стоп-листом и паузой/закрытием точки;
- показывать выплаты (сумма, удержанные 7%, дата).

Один бот на всю платформу, рестораны подключаются по коду — каждому летят только его заказы.

Курьерский бот из проекта исключён по решению от 2026-07-05 — доставка остаётся зоной ответственности ресторана.
